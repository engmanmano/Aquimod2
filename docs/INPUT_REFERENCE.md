# AquiMod2 Input.txt Configuration Reference

This document provides a comprehensive breakdown of configurable options in the `Input.txt` file used by AquiMod2, based on the official AquiMod2 User Manual.

---

## üìÅ Input File Structure

The `Input.txt` file contains a series of keyword-based sections followed by values that control simulation behavior. Each keyword should be followed by its parameter line on the next line.

---

## üîß Keywords and Options

### 1. `Component IDs`

Defines the IDs of the soil, aquifer, and recharge components.

```
Format: <soil_component> <aquifer_component> <recharge_component>
Example: 1 1 1
```

### 2. `Simulation mode`

Determines which simulation mode will be run.

```
Options:
- m ‚Üí Monte Carlo calibration
- s ‚Üí SCE-UA calibration
- e ‚Üí Evaluation mode (runs model using parameter samples)
```

### 3. `Monte Carlo parameters`

Only used if `Simulation mode = m`

```
Format: <number_of_simulations> <acceptance_ratio> <max_iterations> g
Example: 100000 0.7 1000 g
```

### 4. `SCE-UA parameters`

Only used if `Simulation mode = s`

```
Format: <max_iterations> <max_function_evals> <p> <m> g
Defaults: -1 -1 -1 -1 g (to use internally defined optimal values)
```

### 5. `Evaluation parameters`

Only used if `Simulation mode = e`

```
Format: <number_of_parameter_sets> g
Example: 1 g
```

### 6. `Objective function and parameters`

Specifies the performance metric used during calibration.

```
Options:
- 1 ‚Üí RMSE (Root Mean Squared Error)
(Note: future versions may support additional metrics)
```

### 7. `Spin-up period`

Defines the number of initial months discarded to allow model stabilization.

```
Format: <number_of_months>
Example: 6
```

### 8. `Write model output files`

Controls which outputs AquiMod2 writes after simulation.

```
Format: <soil_output_flag> <gwl_output_flag> <component_output_flag>
Each flag is either Y (yes) or N (no)

Order:
1. Soil moisture output (e.g. SM_sim.out)
2. Groundwater level output (e.g. GWL_sim.out)
3. Component output (e.g. *_TimeSeries1.out)

Example: Y Y Y ‚Üí write all output files
```

---

## üìÑ Notes and Conventions

* All flags and parameter values are **case-sensitive**.
* The sequence of sections must remain **as listed in the manual**.
* Output files are placed in the `Output/` folder within the scenario directory.

---

## üìö Common Output Files (if enabled)

Output files generated by AquiMod2 depend on the simulation mode (`m`, `s`, or `e`) and the output flags specified in `Input.txt` (see `Write model output files`). Below is a summary based on Section 5.4 of the AquiMod2 User Manual.

### üîÅ Calibration Modes (`m` or `s`)

* `*_calib.out`: Calibration parameter sets for each component (e.g., `FAO_calib.out`, `Weibull_calib.out`)
* `fit_calib.out`: Objective function values for the parameter sets

Notes:

* The structure of `*_calib.out` matches the `*_eval.txt` format
* No time series outputs (`*_TimeSeries*.out`) are produced in calibration modes

### üìä Evaluation Mode (`e`)

* `*_TimeSeries1.out`: Component time series for each parameter set (e.g., `Q3K3S1_TimeSeries1.out`, `Weibull_TimeSeries1.out`)
* `fit_eval.out`: Objective function values (e.g., RMSE) for each evaluation run

Notes:

* One file per component per evaluation parameter set is generated
* Time series include groundwater level, flow rates, etc.

## üìå Best Practices

* Keep a template `Input.txt` and clone it when starting a new scenario
* Use helper scripts to edit fields programmatically
* Always version control your `Input.txt` files for reproducibility
